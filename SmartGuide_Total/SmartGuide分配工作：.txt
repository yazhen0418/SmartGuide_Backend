分配工作：

🟢 成員 1：AI 視覺與模型位 (大腦)
核心任務：教電腦看懂「台灣的路」

雖然有 NVIDIA 模型，但它像個「剛來台灣的外國人」，看不懂紅綠燈的小綠人或機車格。

具體工作內容：

影像採集： 拿手機固定在胸口（模擬未來掛在身上的高度），在校園走 10 分鐘錄影。

抽幀與標註： 從影片中每秒截一張圖。使用 Roboflow（線上工具，0 基礎友善）把照片中的「路障、階梯、地板」圈出來。

模型微調： 學習如何把這些圈好的台灣照片，餵給模型，讓它辨識率從 70% 提升到 95%。

輸出定義： 決定模型要回傳什麼。例如：{"type": "stairs", "status": "down"}。

需要學習的事項 (從 0 開始)：

Python 環境安裝： 學習安裝 Anaconda 或 Miniconda，管理 Python 套件。

基礎程式： 學習如何用 Python 開啟網路攝影機 (Webcam) 或讀取影片檔。

深度學習概念： 理解什麼是「訓練集 (Training Set)」和「驗證集 (Validation Set)」。

NVIDIA 工具鏈： 學習如何使用 TensorRT 加速模型（搜尋關鍵字：TensorRT Python tutorial）。

🔵 成員 2：硬體通訊與模擬位 (神經)
核心任務：讓 ESP32 假裝自己是「智慧導盲杖」

你現在沒感測器，所以你要負責「寫劇本」給系統演。

具體工作內容：

模擬數據發送： 寫一段程式讓 ESP32 每秒透過 WiFi 傳送一次假數據。

範例：前 10 秒傳「距離 200」，後 5 秒傳「距離 30」，模擬快撞到牆。

硬體互動測試： 練習按一下 ESP32 的按鈕，電腦端就會跳出「啟動辨識」的字樣。

通訊協定： 跟組員 3 討論好 JSON 格式。

格式範例：{"dist": 120, "tilt": 5}（距離 120公分，傾斜 5 度）。

需要學習的事項 (從 0 開始)：

Arduino IDE： 下載並學習如何把第一個 Blink (閃爍燈) 程式燒進 ESP32。

C++ 基礎： 學習變數 (int, float)、判斷式 (if/else) 和迴圈 (loop)。

WiFi 連線： 學習 WiFi.h 函式庫，讓 ESP32 連上你的手機熱點或家裡 WiFi。

JSON 包裝： 學習使用 ArduinoJson 函式庫來排版你要傳出的資料。

🟡 成員 3：後端邏輯與語音位 (總機)
核心任務：當「大腦」跟「神經」打架時，決定誰是對的

你是最重要的一環，因為你要把所有人的東西接起來。

具體工作內容：

建立「數據中心」： 用 Node.js 寫一個伺服器。

接收並顯示： 當成員 1 傳來「看到車」且成員 2 傳來「距離 50cm」時，你的螢幕要能同時顯示這兩行字。

邏輯大師： 寫判斷邏輯。

邏輯：如果 (看到紅燈) 則 (語音播報：請停下)。

語音測試： 串接 Google TTS，讓電腦能根據指令開口說話。

需要學習的事項 (從 0 開始)：

Node.js 安裝： 學習使用 npm 管理套件。

Express 框架： 學習如何建立一個簡單的 API 接收站。

非同步 (Async/Await)： 這是 Node.js 的精髓，保證程式在等 ESP32 資料時，不會讓影像辨識卡住。

Google Cloud 帳號： 學習申請 API Key 並使用 Google TTS SDK。
